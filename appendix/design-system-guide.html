Modes: light, dark

üé® Colors

Semantic Color Tokens

Use these semantic tokens instead of hardcoding colors. They automatically adjust for light/dark mode.

Token Usage Example
--bg-base Main page background background: var(--bg-base);
--bg-surface Cards, elevated elements background: var(--bg-surface);
--text-primary Main text color color: var(--text-primary);
--text-secondary Secondary text, labels color: var(--text-secondary);
--border-default Default borders border: 1px solid var(--border-default);

Brand Color Tokens

Team-specific colors that change based on the active theme:

--brand-core        /* Primary team color */
--brand-light       /* Secondary team color */
--brand-dark        /* Dark team variant */
--brand-interactive /* Interactive elements */

‚úÖ DO: Use CSS Variables

color: var(--text-primary);
border: var(--border-weight-100) solid var(--border-default);
box-shadow: var(--shadow-sheet);

‚ùå DON'T: Hardcode Values

border: 1px solid #E0E0E0;
box-shadow: 0 2px 8px rgba(0,0,0,0.1);

‚òÅÔ∏è Drop Shadows

Shadow Tokens

Two shadow systems for different elevation contexts:

Token Context Use For
--shadow-sheet Mobile Elevated surfaces, cards, bottom sheets
--shadow-modal Tablet/Desktop Modals, dialogs, highest elevation overlays
Note: Each shadow is composed of two layers (A and B) that work together to create depth. Shadows automatically adjust for dark mode.

Shadow Values

/* Sheet Shadow (Mobile) */
--shadow-sheet-a: 0px -8px 64px 0px rgba(0, 0, 0, 0.12)
--shadow-sheet-b: 0px -4px 32px 0px rgba(0, 0, 0, 0.08)
--shadow-sheet: var(--shadow-sheet-a), var(--shadow-sheet-b)

/* Modal Shadow (Tablet/Desktop) */
--shadow-modal-a: 0px 8px 64px 0px rgba(0, 0, 0, 0.16)
--shadow-modal-b: 0px 4px 32px 0px rgba(0, 0, 0, 0.12)
--shadow-modal: var(--shadow-modal-a), var(--shadow-modal-b)

üìã Best Practices

‚úÖ DO

  • Use CSS variables for all colors
  • Use utility classes for spacing
  • Use border and shadow tokens
  • Follow the 8px spacing grid
  • Choose appropriate containers
  • Use text style classes
  • Test in both light and dark modes
  • Respect team-specific button radius

‚ùå DON'T

  • Hardcode color values
  • Hardcode border or shadow values
  • Use arbitrary spacing (13px, 27px)
  • Create custom max-widths
  • Recreate existing text styles
  • Use same container everywhere
  • Ignore responsive behavior
  • Mix shadow contexts (mobile vs desktop)
