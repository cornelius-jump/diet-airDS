List Rows & Cards

Brand badge containers, game matchup headers, tiles, the 3-slot list row architecture, and higher-order patterns built from it: inventory rows, selectable rows, and buy-flow event cards. All built by composing the same base components.

List Rows

3-slot architecture: Leading → Main Content → Trailing. Each slot is optional. Main content stacks text pair, tags, and info blocks vertically. Use .list-row-text-pair (not .card-text-pair) inside list rows.

List row text pair is fixed scale: .labelBold30 label + .labelRegular10 sublabel with 2px gap. Don't use .card-text-pair inside list rows — it has different spacing.

Usage Pattern

Selector

A clickable container that wraps a list row to make it a selectable item. Two variants: wash (nearly transparent at rest — use inside cards or on --bg-surface) and card (shows --bg-surface at rest — use on --bg-base so it looks like a card). Both variants invert to solid black/white when selected and fade to 25% opacity when disabled.

The .selector replaces the padding wrapper that list rows normally need — do not add an extra padding div. The list row sits directly inside .selector with no additional wrapper.

Variant: Wash surface-washNeutral

Nearly transparent at rest. Use inside cards, sheets, or on --bg-surface backgrounds where you want subtle row separation without adding visual weight.

Cards

Two core layout types — Closed (full background, divided sections) and Open (transparent container with separated subsections). Add -interactive suffix for hover effects.

Closed Layout Card

A full-background card with divided sections. Use for structured content like events, products, or details. Background: --bg-surface, Radius: 16px, Sections divided by 1px --neutral-200 borders.

Key rules:

1. .event-row background is intrinsic to the component CSS — no surface token needed on the wrapper.

2. Use surface-section on interactive sections — it provides a transparent hover/press overlay without changing the background color.

3. Never use scale-* on individual sections of a card unit — scale on only part of a card looks wrong.

4. Add .not-tappable to the .list-row inside non-interactive sections (Sold Out, Coming Soon).

Matchup

Two-panel side-by-side layout for game matchup headers. Away team on the left (set --badge-bg inline), home team on the right (inherits --brand-core). Always uses 64px logos — the parent wrapper controls the overall size. The wrapper needs border-radius + overflow: hidden to clip the panel corners correctly.

Tile

Universal small card for grids, carousels, and inside card-open. The .tile wrapper provides the container shape and overflow; .tile-info holds the content area below the visual header.

Interactivity rule — pick one, never both, never mixed in a set:

No button → tile is the tap target → add surface-card scale-700 to .tile

Has button → button is the CTA → no surface-card / scale-700 on .tile

All tiles in a grid or carousel must share the same pattern — never mix tap-target tiles with button tiles in the same row.

Tap Target (no button)

The whole tile is interactive. Add surface-card scale-700 directly to .tile.
