Logo Block

Team-colored tile with a fixed-size centered logo. The container fills its parent — the logo size modifier (sm/md/lg) controls the logo image, not the container.

Logo size variants

The -sm/-md/-lg modifier sets the logo image size only. The colored container fills whatever parent you place it in — all three examples below use the same 200×200 parent.

Small — 64px logo
Medium — 80px logo
Large — 104px logo

All teams — medium logo (120×120 parent)

Each block scopes its own theme via data-theme. Background color comes from each team's --brand-core token.

Matchup

Two panels side by side — away left, home right. Always uses the smallest (64px) logo. Height comes from the parent — wrap the matchup in a div and set height there.

Tile

The recommended pattern for any small card displayed alongside others — event listings, product grids, schedules, content feeds. Tiles work in grids, carousels (especially on mobile), and inside .card-open as content sections. A fixed-height visual header (matchup, image, logo-block) sits above .tile-info. Interactivity rule: no button → add surface-card scale-700 to .tile; has button → button is the CTA, no surface/scale on the wrapper.

Matchup in list row

A matchup can replace the leading image in a list row — desktop only. On mobile, collapse to just the opposing team logo block (same as Section 6). This example uses the large image dimensions (244×124, same as .leading-image-large) with leading-gap-xl — set border-radius and overflow: hidden on the wrapper so panel corners clip correctly.

Opposing teams — CMS color

Opposing teams from Sanity have a primaryColor hex (often null). Set --badge-bg inline to override. When null, fall back to var(--neutral-200) — never leave it unset or it will incorrectly inherit the home team's --brand-core.

HTML pattern

<!-- LOGO BLOCK — parent sets container size, logo-block fills it -->
<div style="width: 80px; height: 80px;">
  <div class="logo-block logo-block-sm">                <!-- home team (uses --brand-core) -->
    <img src="logo.svg" alt="Team Name">
  </div>
</div>
<div style="width: 80px; height: 80px;">
  <div class="logo-block logo-block-sm" style="--badge-bg: ${away.primaryColor ?? 'var(--neutral-200)'};">
    <img src="${away.logoUrl}" alt="${away.name}">      <!-- away team -->
  </div>
</div>

<!-- MATCHUP — parent sets height; always uses smallest (64px) logo -->
<div style="height: 120px;">                           <!-- parent sets height -->
  <div class="matchup">
    <div class="matchup-panel" style="--badge-bg: ${away.primaryColor ?? 'var(--neutral-200)'};">
      <img src="${away.logoUrl}" alt="${away.name}">
    </div>
    <div class="matchup-panel">                        <!-- home team, no --badge-bg needed -->
      <img src="home-logo.svg" alt="Home Team">
    </div>
  </div>
</div>

<!-- TILE — event card with matchup header -->
<div class="tile">
  <div style="height: 120px;">                         <!-- controls matchup height within tile -->
    <div class="matchup">
      <div class="matchup-panel" style="--badge-bg: ${away.primaryColor ?? 'var(--neutral-200)'};">
        <img src="${away.logoUrl}" alt="${away.name}">
      </div>
      <div class="matchup-panel">
        <img src="home-logo.svg" alt="Home Team">
      </div>
    </div>
  </div>
  <div class="tile-tag">Home</div>                    <!-- optional -->
  <div class="tile-info">
    <div class="card-text-pair">
      <span class="labelBold30">vs. Away Team</span>
      <span class="labelRegular10 text-secondary">Sat, Mar 15 · 7:00 PM</span>
    </div>
    <span class="labelBold20 text-success">From $45</span>    <!-- optional -->
    <button class="btn btn-primary btn-100">Buy Tickets</button>  <!-- optional -->
  </div>
</div>
